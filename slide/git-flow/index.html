git-flowとは

masterブランチ

一番メインのブランチ

ココにはリリースされたものだけを管理する

developブランチ

開発用ブランチ

masterブランチが親となる

featureブランチ

機能開発を行うブランチ

機能単位でブランチを切る

developブランチがが親となる

releaseブランチ

リリース前の作業を行うブランチ
(確認・developのバグ修正・バージョンを管理)

確認ができたらmaster / developブランチにマージされる

developブランチがが親となる

hotfixブランチ

リリース後に重大なバグが見つかった時にバグ修正などを行うブランチ

修正が完了したらmasterとdevelopにマージする。
